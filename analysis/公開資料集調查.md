# 公開資料集調查 — 海水小球藻訓練資料

> 為了找能直接拿來「預訓練」模型的公開資料,實際搜尋了 Kaggle、GitHub、論文、Zenodo、PANGAEA、NOAA 等來源。

## ❌ 直接結論:沒有能直接用的

你的設定組合(**海水 + 小球藻 + EC 35-55 + 雙 CO2 + 拍照測濃度**)太特殊,公開資料集都對不上。即使勉強能用的,訓練出來的模型很可能比直接用你自己 5 天資料還差。

---

## 各來源評估

### 1. Kaggle
- **Algae Bloom Prediction**(UCI 河流):https://www.kaggle.com/datasets/uciml/algae-blooms
  - 河水化學數據 + 藻類屬,**淡水、無時序、缺 EC/lux/CO2**。當參考用,不能訓練。
- **Harmful Algal Bloom 系列**:沿岸衛星/浮標資料。**海洋大尺度,不是實驗室**。
- 沒有專做小球藻實驗室培養的 Kaggle 資料集。

### 2. GitHub
- **`bryanoliveira/microalgae-growth-prediction`**:https://github.com/bryanoliveira/microalgae-growth-prediction
  - 有小球藻屬資料 + 光、溫、養分,但**淡水**、無濁度、時間粒度差太多。
  - **可以參考它的 LSTM / regression 模型架構**,資料不能直接用。
- 其他 hobby repo 都是玩具資料集,純參考。

### 3. 論文 + 補充資料
- 近年 **Sharma 2024、del Rio-Chanona 2019** 等論文都有用 ANN/RF 訓練小球藻生長模型
  - 但補充資料多是**端點測量**(n=50~300),不是連續時序
  - 多數論文發表的是**擬合好的模型參數**(μ_max, K_s),不是原始感測器資料
  - Google Scholar:https://scholar.google.com/scholar?q=chlorella+machine+learning+growth+prediction
- 用法:**抄參數**而不是訓練模型

### 4. Zenodo / FigShare / Dryad
- Zenodo 搜「Chlorella cultivation」:https://zenodo.org/search?q=chlorella+cultivation
  - 多數是「單變數對應終點生物量」,沒有完整感測時序
- PANGAEA:海洋實地資料豐富,但實驗室培養資料少
- 都只能當參考

### 5. NCEI / NOAA Chlorophyll-a
- 衛星/浮標的海洋葉綠素 a 資料
- **公里尺度 vs 你燒杯尺度,完全對不上**,跳過

---

## 為什麼公開資料用不上你

1. **領域不符**:海水離子組成 + 封閉系統 CO2 動態 + 拍照測濃度,這組合在公開資料中沒有
2. **尺度不符**:現場/衛星 → 實驗室小規模,模型無法遷移
3. **變數不符**:很少資料同時記錄 lux + EC + 雙 CO2
4. **標籤不符**:你的「拍照算濃度」是自製代理,不是標準 OD750 或細胞計數

## ✅ 實際可行的策略

**主訓練資料用你自己的 5 天**,配合**文獻給的生長方程式**作骨架。

### 1. 基線模型:Monod + 光限制
這是水產文獻通用的小球藻生長公式:
```
μ = μ_max × (S / (K_s + S)) × f(I)
```
其中 `f(I)` 是光限制函數(Steele 或 Eilers 模型,光太多/太少都會限制生長)。

**μ_max 從你自己的資料擬合**,**K_s 從文獻給的參考範圍找一個合理值**:
- 海水小球藻典型 μ_max ≈ **0.5 ~ 1.5 day⁻¹**
- 換算成每小時:**0.02 ~ 0.06 hr⁻¹**

### 2. ML 加層(可選):Random Forest 補殘差
讓 Monod 預測一個基準值,**ML 學「實際 - Monod」的殘差**:
- 你 5 天資料 ≈ 600 筆小時級資料,**夠用 5 個 feature 的 Random Forest**
- LSTM / Transformer 在這個資料量下會 overfit,**不要用**

### 3. 文獻是查參數的工具,不是訓練資料
- 上面 GitHub repo 的程式碼結構可以借鏡
- Sharma 系列論文找 hyperparameter 的合理區間
- 花 1 小時翻翻就好,不要花幾天搞資料工程

## 如果你還是很想要更多資料

**推薦的折衷做法**:
1. 比賽 5 天**故意製造不同條件**(光照變化、CO2 變化、溫度變化)→ 等於 5 個小實驗
2. 比賽結束後,把當天的條件 + 對應生長率,當成 ~5 個獨立資料點
3. 配合文獻的 μ_max 範圍當先驗,做貝氏迴歸(PyMC、Stan)
4. 比訓練深度神經網絡更適合你的資料量

**真要硬找,我推**:`bryanoliveira/microalgae-growth-prediction`(GitHub),學它的 RF 結構,資料另用自己的。

---

## 結論

**省下找資料的時間,把心力放在**:
1. 校準拍照測濃度的方法
2. 5 天實驗變化條件設計
3. 用 sklearn / Random Forest + Monod 公式做混合模型
4. AI 報告(Claude API)做加分項

公開資料集這條路投資報酬率太低。
