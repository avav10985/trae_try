## 兩個問題分開看

## 為什麼純文字救不了 `new Date()`

純文字格式的「保護」只擋**使用者鍵盤輸入時**的自動解析(防止你打 `04:00` 被改成 `上午 4:00`)。

但 Apps Script 的 `appendRow([new Date(), ...])` 是把一個 **JS Date 物件**丟給 Sheets。Sheets 收到 Date 物件後,**為了存進文字格的儲存格**,會先用該 Sheet 的 locale 把它轉成字串再存。

結果通常會是這兩種其中一種(看 Sheet 的時區/地區設定):
- `Wed Apr 29 2026 06:00:00 GMT+0800 (Taipei Standard Time)` ← JS 預設 toString
- `2026/4/29 上午 6:00:00` ← Sheets locale 轉換

**兩種 pandas 的 `pd.to_datetime` 都認不出來**(`errors='coerce'` 直接吃成 NaT → 整列被丟)。

## 最保險的做法

改 `googlecsv.js` 那一行,**直接強制輸出 ISO 格式**:

```js
Utilities.formatDate(new Date(), "Asia/Taipei", "yyyy-MM-dd HH:mm:ss"),
```

這樣不管 Sheet 的地區怎麼設、欄位是什麼格式,寫進去的就是 `2026-04-29 06:00:00` 純字串,pandas 100% 認得。

## `config.py` 是什麼

[`analysis/config.py`](analysis/config.py) — **整個 `analysis/` 資料夾的中央設定檔**。它沒有「執行邏輯」,只是一堆變數定義,讓你**改設定只動一個檔**而不用每支程式翻來改去。

### 它管的東西分 6 類

| 類別 | 變數 | 控制什麼 |
|---|---|---|
| **資料來源** | `CSV_FILE` | Pi 寫 CSV 的路徑(`~/Desktop/algae_monitor_data.csv`)— 所有讀取程式都從這裡讀 |
| **輸出位置** | `ALERT_LOG`, `REPORT_DIR` | 警告 log 跟每日報告 HTML 寫到哪 |
| **欄位定義** | `SENSOR_COLS` | 9 個感測器欄名(必須跟 Pi 寫的 CSV 標題一致,**改了會炸**) |
| **異常閾值** | `HARD_LIMITS` | 9 個感測器的硬上下限(超出觸發警告) |
| **監測參數** | `CHECK_INTERVAL_SEC`, `ZSCORE_WINDOW_HOURS`, `ZSCORE_THRESHOLD`, `DISCONNECT_WINDOW_MIN` | 多久檢查一次、Z-score 怎麼算、斷幾分鐘判定脫線 |
| **Email 設定** | `EMAIL_ENABLED`, `EMAIL_SENDER/PASSWORD/RECEIVER`, `EMAIL_SUBJECT_PREFIX` | Gmail SMTP 寄信設定(就是你問的那段) |

### 誰會用到它

```
config.py
   ↑ from config import ...
   │
   ├── monitor_email.py     → 讀 HARD_LIMITS, ZSCORE_*, CSV_FILE, EMAIL_*
   ├── daily_report.py      → 讀 CSV_FILE, REPORT_DIR, SENSOR_COLS, FONT_FAMILY
   ├── ai_report_email.py   → 讀 CSV_FILE, REPORT_DIR, SENSOR_COLS
   └── email_helper.py      → 讀 EMAIL_*(只負責寄信)
```

`dashboard.py` **沒用** `config.py`(因為要部署到 Streamlit Cloud,設定改用 secrets,自己內建一份 SENSOR_COLS / HARD_LIMITS — 這是個小重複,但保留是因為部署的對象不一樣)。

## 1. Commit 是什麼?用 GitHub Desktop 還需要我做嗎?

**Commit 跟 GitHub Desktop 做的事完全一樣**,只是介面不同。

| 動作 | 我在 terminal 做 | GitHub Desktop 做 |
|---|---|---|
| 看哪些檔改了 | `git status` | 左側「Changes」自動列出來 |
| 選哪些檔要打包 | `git add 檔名` | 勾選核取方塊 |
| 打包 + 寫說明 | `git commit -m "說明"` | 下面填 Summary → Commit |
| 推到 GitHub | `git push` | 上方藍色「Push origin」 |

**結論:你用 GitHub Desktop 就夠了,我不用 commit**。我修改的檔案已經存在硬碟上,你打開 GitHub Desktop 會自動看到所有變動,你自己選要不要 commit 跟推上去。

## 系統架構

整個系統分成 **3 個區段**,每段在不同地方運作、做不同的事:

### 🔬 感測端 — 「在水裡量」
**位置:** 養殖桶旁邊
**硬體:** Arduino UNO + 9 顆水質感測器
**做什麼:** 每 10 秒量一次水的溫度、pH、含鹽量、濁度、CO₂、光照……,然後把數據透過 USB 線傳給樹莓派
**主程式:** [`uno_sensor_serial.ino`](uno_sensor_serial/uno_sensor_serial.ino)(燒進 Arduino 一次,之後它自己跑)

### 🍓 本地端 — 「在家裡的小電腦處理」
**位置:** 家裡,接著 Arduino + 接著 Wi-Fi
**硬體:** Raspberry Pi(樹莓派,信用卡大小的電腦)
**做什麼:** 接收 Arduino 量到的數據,然後做兩件事:
  - **① 存一份在桌面**(本地備份,即使網路掛了也不會掉資料)
  - **② 把同一筆送上雲端**(讓你出門也能看)

**主程式:** [`rpi_gui_monitor.py`](rpi_gui_monitor.py)(Pi 開機就要跑)

**附加功能**(可選,也在 Pi 上跑):
| 程式 | 做什麼 |
|---|---|
| ✉ [`monitor_email.py`](analysis/monitor_email.py) | 數據異常 → 立刻寄 email 通知你 |
| 📊 [`daily_report.py`](analysis/daily_report.py) | 每天自動產生圖表 HTML 報告(本地檔) |
| 🤖 [`ai_report_email.py`](analysis/ai_report_email.py) | 每天請 Claude AI 寫一段文字日報並寄到信箱 |

### ☁ 線上端 — 「網路上看得到」
**位置:** Google 和 Streamlit 公司的伺服器(**不是**你的電腦)
**做什麼:** 把雲端那份資料變成漂亮的網頁,**任何人有網址都能看**(電腦、手機、平板都行)
**靠的是兩個免費服務串起來:**
  - **Google 試算表** — 像 Excel 但在雲端,Pi 把數據傳這裡;[`googlecsv.js`](googlecsv.js) 是接收程式
  - **Streamlit Cloud** — 把試算表變成有圖表、4 個分頁的網頁;[`dashboard.py`](analysis/dashboard.py) 是網頁本身

**目前網址:** `https://aquamind-algae.streamlit.app`
