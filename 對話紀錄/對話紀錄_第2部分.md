## 短答:校準程式本身不需要,但有的話更穩

### 為什麼校準時不一定要接溫度感測器
官方流程是基於兩個假設:
1. **緩衝液在 25°C 時 pH 才等於標示值**(pH 4.00 / 7.00 / 9.18 都是 25°C 的標稱值)
2. 校準環境就是室溫,所以直接用標稱值算 Offset 即可

換句話說,只要你校準時水溫在 **20~30°C 之間**,直接套用 7.00、4.00 去算誤差就夠準了,不用接 DS18B20。

### 不同緩衝液對溫度的敏感度

## 這是正常現象,因為 Offset 和 Gain 會互相影響

### 為什麼會這樣
公式是 `pH = 3.5 × V + Offset`
- `Offset` 控制「整條線上下平移」
- `3.5`(硬體 gain 電位器) 控制「整條線傾斜度」

當你轉電位器調 pH 4 時,整條線是**繞著某個點轉動**,不是平移。所以 pH 7 那邊的讀值幾乎一定會跟著跑掉。

```
理想情況: 兩條線交於 pH 7 和 pH 4 兩點
實際情況: 調 pH 4 → pH 7 跑掉 → 調 pH 7 → pH 4 又跑掉一點
```

### 解法 A:迭代調整(官方流程的隱含做法)
不要期待一次到位,要來回幾次:

1. pH 7 → 改 Offset → 讀值對齊 7.00
2. pH 4 → 轉電位器 → 讀值對齊 4.00
3. **回去測 pH 7** → 如果偏 6.85,把 Offset 再加 0.15
4. **再回去 pH 4** → 偏 4.10,電位器**微微**逆轉一點
5. 重複 3、4,直到兩邊都在 ±0.1 以內(通常 2~3 輪就收斂)

**訣竅**:每次調電位器時轉**很小一格**,不要一次轉很多。

### 解法 B:放棄硬體 gain,改用純軟體兩點校準(更可靠)
這樣兩個參數是用數學算出來的,不會互相干擾。流程:

1. 電位器**先不要動**(維持出廠位置)

4. 算:

5. 主程式改成 `pH = slope * V + offset`,一次搞定

## 8 我想怎麼改 — 解釋

**現在的死碼**(rpi_gui_monitor.py:168-177):
```python
final_data = {}
for key in self.sensor_keys:
    val = v.get(key)
    if val is not None:           # ← UNO 永遠送全部欄位,所以 val 不可能 None
        final_data[key] = val
        if self.status[key].get():
            self.data_vars[key].set(f"{val}")
    else:                          # ← 死碼,永遠走不到
        final_data[key] = "---"
        self.data_vars[key].set("無數據")
```

**問題**:
- `val == -1` 時(感測器斷線),GUI **直接顯示 "-1"**,使用者看到一串 -1 不一定知道是斷線
- 如果未來 UNO 韌體真的少傳某欄位,整個程式靜默把字串 `"---"` 塞進 CSV 和雲端,反而會壞事

**我提議改成**:
```python
final_data = {}
for key in self.sensor_keys:
    val = v.get(key)
    if val is None:                            # 韌體沒送這欄(理論上不會發生)
        final_data[key] = -1                   # 統一視為斷線
        if self.status[key].get():
            self.data_vars[key].set("⚠ 未送")
    elif val == -1:                            # 感測器確認斷線
        final_data[key] = -1
        if self.status[key].get():
            self.data_vars[key].set("⚠ 未接")
    else:                                      # 正常讀值
        final_data[key] = val
        if self.status[key].get():
            self.data_vars[key].set(f"{val}")
```
